# 音频 THD 实验准备完成记录（2026-06-30）

## 1. 这次补的不是“结果”，而是实验底座

前面我已经确认了一个事实：

- 当前项目资料里没有现成的标准 `1kHz` 正弦波样本
- 也没有一份已经做完的板端 THD 报告

如果停在这里，我只能一直说“以后再测”。  
所以这次我继续往前推进了一步：把做严谨 THD 实验所需要的**标准输入样本生成工具**和**分析工具**都补齐。

---

## 2. 我新增了什么

### 2.1 标准正弦波生成脚本

- `tools/generate_sine_wave.py`

这个脚本可以直接生成标准 WAV：

- 可指定频率
- 可指定采样率
- 可指定时长
- 可指定单声道 / 双声道
- 可指定幅度

这样后面要做 THD 实验时，我不需要再去临时找外部样本。

### 2.2 音频质量分析脚本

- `tools/analyze_audio_quality.py`

这部分前面已经补过，作用是：

- 统计音频基本参数
- 估计主频
- 对接近单频正弦波的输入给出粗略 THD
- 对宽带/复杂音频给出“不适合作为 THD 指标”的提示

---

## 3. 这一步为什么有价值

虽然我现在还没有板端“采进去再录回来”的标准正弦波结果，但我已经把实验前最容易拖延的两件事解决了：

1. **标准输入怎么来**
2. **录完之后怎么分析**

这意味着后面如果我要正式把 `THD < 5%` 做实，流程已经非常明确：

1. 用脚本生成标准 `1kHz` WAV
2. 让它进入板端采集/编码链路
3. 拿回结果音频
4. 用分析脚本看主频、谐波和 THD

---

## 4. 当前能讲到什么程度

这一步之后，我现在能更严谨地说：

- 我不仅知道 `THD < 5%` 该怎么测
- 我已经把实验输入样本生成工具和分析工具准备好了

但我还不会把它写成“当前工程已达成”。

因为真正严谨的结论还需要：

- 把标准正弦波送入真实板端链路
- 再采回板端输出音频
- 最后对输出做频谱分析
