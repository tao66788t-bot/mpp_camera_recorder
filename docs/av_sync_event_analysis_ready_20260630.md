# AV Sync 事件级后处理链补充记录（2026-06-30）

## 1. 目标

前面项目里对 `AV sync` 只有轨道级粗验证：

- 比较音视频 track 时长
- 看容器层是否存在大偏移

这只能说明“没有明显跑飞”，还远远不够支撑：

- `AV sync < 30ms`

所以这次继续补了一条更强的后处理链，目标是：

- 对“闪光 + 蜂鸣”或“拍手”一类受控事件 MP4
- 自动估算视频事件时刻
- 自动估算音频事件时刻
- 输出事件级音视频偏差

---

## 2. 新增脚本

- `tools/analyze_av_sync_event.py`

核心思路：

1. 逐帧解码视频，计算每帧平均亮度
2. 找到最大亮度突变位置，作为视频事件点
3. 从同一 MP4 中提取音频为 WAV
4. 计算短窗 RMS 包络，找到最大包络突变位置，作为音频事件点
5. 输出 `audio_minus_video_ms`

---

## 3. 本机环境结论

这条后处理链依赖的本机能力已经确认可用：

- `cv2`
- `imageio`
- `imageio_ffmpeg`
- `numpy`

其中 `imageio_ffmpeg` 还顺带解决了本机系统 `PATH` 里没有完整 ffmpeg 的问题。

---

## 4. 当前意义

到这一步为止，我虽然还没有拿真实“闪光 + 蜂鸣”板端样本去跑出 `<30ms` 的最终数字，但我已经把后处理链补到了：

- 录到一段受控事件 MP4
- 就能在本机跑出事件级偏差估计

也就是说，`AV sync` 现在和 `THD` 一样，剩下的主要缺口已经不是工具链，而是一段标准化板端采样。

---
