# IPC 项目简历安全版条目（2026-06-30）

## 1. 这份文档的用途

这份文档不是继续堆项目名词，而是把当前这份 V853 IPC 工程里**已经被代码、样本文件、运行日志或分析脚本证明过**的内容，整理成一版可以直接贴进简历的安全条目。

我的目标是：

- 保留真正做实的成果
- 去掉会被代码细节追问击穿的模板数字
- 让简历里的每一句话都能回到项目证据

---

## 2. 可直接放进简历的版本

### 版本 A：两到三行浓缩版

> 基于全志 V853 + MPP 搭建 IPC 固件主链路，完成 `GC2053 -> VI -> LCD(480x800) / H.265(1920x1088@20fps)` 的本地预览与录制，板端稳定输出 MP4 与 RTSP。  
> 打通 `AI -> AENC -> AAC -> MP4/RTSP` 音频链路，并在 Windows 端通过 VLC 验证 RTSP 音视频联调。  
> 结合 SDK sample 与离线分析脚本优化 LCD 预览和 H.265 码流稳定性，将 MP4 帧大小离散系数 `CV` 从 `143.70%` 降到 `92.70%`。

### 版本 B：更适合校招/实习简历的 4 点版

> **全志 V853 IPC 音视频固件开发**  
> - 基于全志 MPP 完成 `GC2053` 采集链路适配，打通 `VI0 -> VENC -> MUX -> MP4` 与 `VI2 -> VO -> LCD` 双视频链路，实现 `1920x1088@20fps` 编码录制与 `480x800` LCD 本地预览。  
> - 接入 `AI -> AENC -> AAC` 音频链路，实现音频同时进入 `MP4` 与 `RTSP`，Windows 端可通过 VLC 观看/收听板端 `rtsp://127.0.0.1:8554/stream` 转发结果。  
> - 参考全志 SDK sample 补齐 H.265 CBR 的 `QP / VBV / RC 帧率` 参数，使用离线脚本统计 MP4 帧大小离散系数，实测 `CV` 从 `143.70%` 优化到 `92.70%`，目标码率保持约 `2Mbps`。  
> - 通过 VI 抓帧、格式对比和离线图像一致性分析定位 LCD 偏色问题，确认 `VI2 NV12 -> VO` 为正确预览路径，修复后预览图与正确参考图像素级一致。

## 4. 现在不能直接写进简历的数字

下面这些数字暂时不要写死到简历里：

- `44.1kHz / 双声道 / 128kbps`
- `色彩还原度提升 20%`
- `音频总失真率 < 5%`
- `MP4 封装速度 +40%`
- `I/O 次数 -60%`
- `音视频同步偏差 < 30ms`

原因不是它们一定错，而是**当前这份工程还没有把它们全部做成可闭环验证的结果**。

## 5. 当前最稳妥的讲法

如果面试官继续往下问，我建议统一按这个思路讲：

1. 我先把 LCD、本地 MP4、RTSP、音频、检测这些主链路全部打通。
2. 然后我专门做了一轮量化项审计，把旧文档里的模板数字和当前工程真实实现分开。
3. 我保留了已经做实的数字，比如：
   - `480x800`
   - `1920x1088@20fps`
   - `RTSP 554`
   - `16kHz/16bit/mono AAC`
   - `CV 143.70% -> 92.70%`
4. 对像 `THD < 5%`、`色彩提升 20%` 这种还没做标准实验闭环的数字，我不会硬写成结果。

这套说法反而更像真做过项目，而不是背了一篇项目包装稿。
