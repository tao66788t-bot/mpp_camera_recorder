# AV Sync 事件级后处理链验证记录（2026-06-30）

## 1. 这次做了什么

为了把 `AV sync < 30ms` 从“只有轨道级粗验证”继续往前推进，我把新补的事件级分析脚本实际跑了一遍：

- `tools/analyze_av_sync_event.py`

验证样本使用的是：

- `sample_mp4_full_verified_20260628.mp4`

注意，这份 MP4 不是专门拍摄的“闪光 + 蜂鸣”受控样本，我这次的目的不是证明 `<30ms`，而是验证：

- 事件级后处理链本身能否真实跑通

---

## 2. 实际输出结果

脚本输出了完整结构化结果，核心字段如下：

- `frame_count = 1950`
- `fps ≈ 19.828`
- `video_event_time_sec ≈ 0.1513`
- `audio_event_time_sec ≈ 0.000125`
- `audio_minus_video_ms ≈ -151.175`
- `interpretation = event-level offset is large or event detection is unstable`

---

## 3. 如何解释这个结果

这个结果**不能**被拿去写成：

- `AV sync < 30ms`

原因不是工具坏了，而是样本不对：

- 这份 MP4 不是受控事件样本
- 没有明确的“闪光 + 蜂鸣”或“拍手”基准事件
- 所以脚本只能在普通画面和普通音频里硬找“最像事件”的跳变点
- 得到的偏移大，很正常

这反而证明了一件重要的事：

- 事件级后处理链已经能跑通，并且会对“不适合做事件同步结论的样本”给出保守解释

---

## 4. 当前真正剩下的缺口

到这一步，`AV sync` 离真正做实，只差：

- 录一段标准化的“闪光 + 蜂鸣”或“拍手”板端 MP4

然后直接用这条链分析，就能得到比当前粗 track-duration 方法更强的同步偏差估计。

也就是说，现在缺的已经不是分析脚本，而是一次规范化采样。

---
